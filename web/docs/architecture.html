아키텍처 개요

현재 구조는 `backend/`, `web/`, `tests/`, `api/`를 제품 축으로 두고, 운영 제어는 `scripts/`, 문서 기준은 `docs/`, 로컬 산출물은 `.workspace/`로 격리하는 형태입니다. CooCook 같은 서비스 페이지는 정적 HTML/JS와 taxonomy 기반 클라이언트 상태로 동작합니다.

프론트엔드

정적 HTML과 공용 JS가 Vercel 정적 출력으로 배포됩니다.

API 연결

`/api/*`는 proxy 경유로 백엔드로 전달됩니다.

문서 구조

정식 기준 문서는 `docs/INDEX.md`에서 시작하며, status/reference/runbooks/checklists로 분리됩니다.

운영 원칙

루트는 진입점만 유지하고, 로그·임시 파일·레거시 보고서는 루트에 두지 않습니다.

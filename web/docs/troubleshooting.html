문제 해결

배포 후 가장 먼저 확인할 것은 404, 잘못된 상대경로, 그리고 클라이언트 스크립트의 DOM 참조 오류입니다. 이번 정리에서는 실제 배포 링크 크롤링으로 누락 경로를 찾고, 정적 문서 링크와 비디오 라이브러리 누락을 수정했습니다.

404가 날 때

정적 파일이 실제로 존재하는지, 경로가 절대/상대 기준에 맞는지, Vercel rewrite와 충돌하지 않는지 확인합니다.

화면이 비어 보일 때

`getElementById` 대상이 실제 DOM에 있는지 확인합니다. DOM ID 불일치는 404처럼 보이는 고장을 만들 수 있습니다.

CooCook이 느려질 때

국가/메뉴/식재료/카테고리 조합이 늘면 단순 배열 순회 대신 taxonomy 인덱스와 캐시 조회를 사용해야 합니다.

## Context
> What is the issue that we're seeing that is motivating this decision or change?

CooCook needs to deliver MVP quickly while maintaining architectural integrity. Team is small (3-5 devs). We need a structure that prevents spaghetti code but doesn't over-engineer with premature microservices.

---

## Decision
> What is the change that we're proposing and/or doing?

Use Clean Architecture (Domain → Use Cases → Interface Adapters → Frameworks) within a single deployable unit (modular monolith). Bounded Contexts map to Python packages. Extract to microservices when a context exceeds 2 devs or 10k req/min.
